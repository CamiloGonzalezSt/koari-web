¿Qué es este sistema?

En pocas palabras: es su propio sistema de pedidos online, sin pagar comisiones a nadie.

Lo que tiene el local ahora

  • Un sitio web donde los clientes ven el menú con fotos y precios
  • El cliente arma su pedido y lo envía por WhatsApp con un solo clic
  • El pedido llega completo y ordenado al teléfono del local
  • Sin apps de delivery, sin comisiones, sin mensualidades

Lo que NO necesita el local

  • No necesitan contratar ningún servicio externo
  • No hay que instalar nada en los teléfonos del local
  • No hay que aprender ningún programa nuevo para recibir pedidos
  • Los pedidos llegan a WhatsApp, igual que siempre
Resumen: El cliente entra al sitio web, elige lo que quiere comer, y le llega un mensaje de WhatsApp al local con todo el detalle del pedido listo para confirmar.

Paso a paso del cliente

  1. 1
    Entra al sitio web Ve el menú completo con fotos, precios y descripciones de todos los productos.
  2. 2
    Agrega al carrito Con el botón "+ Agregar". Si el producto tiene opciones (rellenos, tamaños), se abre una pantalla para elegir.
  3. 3
    Abre el carrito y elige cómo quiere recibirlo Delivery (a domicilio) o Retiro en el local. Si elige Delivery, elige la comuna y aparece el costo de envío automáticamente.
  4. 4
    Completa su nombre y teléfono Son los únicos campos obligatorios. Puede guardarlos para no escribirlos la próxima vez.
  5. 5
    Elige cómo va a pagar Efectivo, transferencia o tarjeta. Si elige transferencia, aparecen los datos bancarios del local automáticamente.
  6. 6
    Clic en "Pedir por WhatsApp" Ve un resumen final de su pedido para revisar antes de enviar.
  7. 7
    Confirma y se abre WhatsApp El mensaje con todo el detalle del pedido ya está escrito. Solo tiene que presionar "Enviar".

El mensaje siempre incluye

  • Lista de productos con cantidades y precios
  • Subtotal + costo de envío + total final
  • Nombre y teléfono del cliente
  • Tipo de pedido (Delivery o Retiro)

  • Fecha y hora si es pedido programado
El local solo tiene que responder el mensaje para confirmar el pedido. No hay que aprender ningún programa nuevo.

Todo lo que puede hacer el cliente

El sitio tiene varias funciones pensadas para que el cliente pida más fácil y más seguido.

🔍 Buscar productos

El cliente escribe cualquier palabra (ej: "salmón") y el menú filtra automáticamente mientras escribe, sin recargar la página.

♥ Guardar favoritos

Puede marcar productos con el corazón para guardarlos. Con el botón de filtro puede ver solo sus favoritos.

🔄 Repetir último pedido

La próxima vez que entra al sitio, puede repetir el pedido anterior con un solo clic. Muy útil para clientes habituales.

📅 Programar para más tarde

Puede elegir una fecha y hora de entrega específica. El sistema solo muestra horarios disponibles del local (no deja elegir domingos ni horarios fuera de atención).

💾 Recordar sus datos

Si marca "Recordar mis datos", la próxima vez que entre su nombre, teléfono y dirección ya están escritos. Se borran automáticamente después de 30 días.

↗ Compartir productos

Puede compartir cualquier producto por WhatsApp, Instagram o como enlace copiado. Bueno para cuando le preguntan qué pidió.

Pedidos programados — con restricción de horario

Si el cliente activa "Programar pedido", el sistema solo le deja elegir fechas y horas en las que el local atiende:

  • No puede elegir domingo (el local no atiende ese día)
  • No puede elegir horas antes de las 12:15 ni después del horario de cierre
  • El horario permitido aparece automáticamente debajo del campo de hora
  • Si elige mal, el campo se pone rojo con un aviso claro

Cómo actualizar el menú

Todo el menú se maneja desde un solo archivo de texto. No es necesario saber programar, pero sí hay que tener cuidado con el formato.

El archivo a editar: sushinan/data/products.json
En este archivo está todo: los productos, precios, categorías, costos de envío, datos bancarios y horarios del local.

¿Qué se puede cambiar?

Qué quiero cambiarDónde está en el archivo
Precio de un productoEl campo "precio" del producto. Solo números, sin puntos ni comas (ej: 5500)
Nombre de un productoEl campo "nombre"
Descripción de un productoEl campo "descripcion"
Marcar como agotadoAgregar "disponible": false al producto
Reactivar un producto agotadoCambiar a "disponible": true o eliminar ese campo
Costo de envío por comunaEn "costos_envio", cambiar el número de la comuna
Datos bancarios para transferenciaEn "pago_transferencia"
Horario del localEn "horario" (texto del footer) y en "horarios_pedido" (horarios reales para validar pedidos)
Teléfono de WhatsApp donde llegan los pedidosEn "telefono_whatsapp" (solo números, sin + ni espacios)

Agregar un producto nuevo

  1. 1
    Abrir el archivo Buscar sushinan/data/products.json y abrirlo con el bloc de notas o VS Code.
  2. 2
    Encontrar la categoría Buscar el texto "id": "rolls-carta" (u otra categoría) para ubicarse.
  3. 3
    Agregar el nuevo producto Al final de la lista de productos de esa categoría, agregar una coma después del último y luego el nuevo:
{ "id": "roll-nuevo-001", ← debe ser único, sin espacios "nombre": "Roll Especial", "descripcion": "Salmón, palta, queso crema", "precio": 5500, ← solo números, sin puntos ni comas "imagen": "img/optimized/rolls/roll-especial.jpg" }
  1. 4
    Agregar la foto del producto Ponerla en la carpeta img/optimized/ dentro de la categoría correspondiente. Tamaño mínimo recomendado: 600×360 píxeles.
  2. 5
    Verificar que no hay errores Correr el comando node scripts/validar-catalogo.js — si hay algún error de formato, lo detecta antes de publicar.

Marcar un producto como agotado

Agregar "disponible": false al producto. En el sitio aparecerá con la etiqueta "AGOTADO" y el botón desactivado. Para reactivarlo, cambiar a true o borrar esa línea.

{ "id": "promo-1", "nombre": "Promo 1 — 20 Piezas", "precio": 7500, "disponible": false, ← agrega esta línea para marcarlo agotado "imagen": "img/..." }
Cuidado con el formato: El archivo JSON es muy estricto. Si falta una coma o un signo de comillas, el sitio deja de funcionar para todos los clientes. Siempre correr el validador (node scripts/validar-catalogo.js) antes de publicar un cambio.
7

Cómo publicar cambios al sitio

Después de editar cualquier archivo, hay que publicarlo para que los cambios se vean en el sitio real. Son 3 comandos simples.

  1. 1
    Validar que todo está bien node scripts/validar-catalogo.js
    Si aparece "✓ Todo OK", continuar. Si hay errores, corregirlos primero.
  2. 2
    Guardar los cambios en Git git add . && git commit -m "descripción del cambio"
    Ejemplo: git commit -m "subo precio promo 1 a $8.000"
  3. 3
    Publicar git push
    Listo. En aproximadamente 30 segundos el sitio web ya muestra los cambios.
¿Cómo sé que se publicó? Entrar al panel de Vercel (vercel.com) o simplemente recargar el sitio web desde el teléfono. Si el cambio se ve, está publicado.

Si quiero cambiar solo un precio

Es exactamente el mismo proceso: editar el número en el JSON, validar, commit y push. En menos de 2 minutos el precio nuevo ya aparece en el sitio para todos los clientes.
